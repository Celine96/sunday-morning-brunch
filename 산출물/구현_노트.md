# 구현 노트 (Developer Agent)

## 기술 스택 및 구조

### 백엔드: `/backend/`
- **프레임워크**: FastAPI (Python 3.9)
- **DB**: SQLite (`backend/data.db`) + SQLAlchemy ORM
- **LLM**: Anthropic Claude API (Haiku: 감성분류, Sonnet: 대댓글 생성)
- **구조**:
  - `app/main.py` — FastAPI 앱 엔트리포인트
  - `app/database.py` — DB 연결 설정
  - `app/models.py` — SQLAlchemy 모델 (Product, Review, Reply, ToneProfile, ReplyHistory)
  - `app/schemas.py` — Pydantic 스키마
  - `app/seed.py` — CSV/JSON 데이터 로드 스크립트
  - `app/llm_service.py` — Claude API 연동 (감성분류 + 대댓글 생성)
  - `app/routers/shop.py` — 쇼핑몰 CRUD API
  - `app/routers/agent.py` — 에이전트 API (톤 설정, 생성, 확정, 게시)

### 프론트엔드: `/frontend/`
- **프레임워크**: Next.js 14 (App Router) + TypeScript + Tailwind CSS
- **구조**:
  - `src/app/shop/` — 샘플 쇼핑몰 (상품 목록, 상품 상세+리뷰)
  - `src/app/dashboard/` — 에이전트 대시보드 (톤 설정, 리뷰 관리, 히스토리)
  - `src/components/` — 공용 컴포넌트
  - `src/lib/api.ts` — API 클라이언트

## 주요 구현 결정사항

### 1. LLM 폴백 처리
- `ANTHROPIC_API_KEY` 미설정 시 폴백 로직으로 동작
  - 감성분류: 키워드 기반 휴리스틱 분류
  - 대댓글 생성: 감성별 템플릿 대댓글 반환
- API 키 설정 시 Claude Haiku(감성분류) + Claude Sonnet(대댓글 생성) 사용

### 2. 데이터 모델
- 아키텍처 문서의 스키마를 그대로 구현
- `Reply.review_id = 0`: 대시보드에서 직접 입력한 리뷰(DB에 저장된 리뷰가 아닌 경우)

### 3. 톤 프로필
- `brand_persona.json` + `brand_tone_presets.json`의 emotional 프리셋을 기본값으로 시드
- 시스템 프롬프트에 브랜드명, 슬로건, CS 원칙, 톤 키워드, 샘플 대댓글 포함
- 감성별 대댓글 전략: 긍정(감사+가치강화), 부정(공감+사과+해결), 문의(답변+안내)

### 4. FAB 미니패널
- 쇼핑몰(`/shop/*`) 페이지에서만 렌더링
- `position: fixed; z-index: 9999`로 오버레이
- 현재 상품의 대댓글 미작성 리뷰만 필터링 (상품 상세 페이지인 경우)
- 단건 대댓글 생성/수정/확정/게시 지원

### 5. 게시 흐름
- "게시" 클릭 시 에이전트 API가 Reply 상태를 `published`로 변경
- 동일 DB이므로 쇼핑몰에서 즉시 반영됨 (별도 API 호출 시뮬레이션)
- 히스토리에 생성/수정/확정/게시 각 단계가 자동 기록

### 6. 보안
- ANTHROPIC_API_KEY는 환경변수로만 처리 (하드코딩 없음)
- CORS는 localhost:3000만 허용
- 입력값 검증: Pydantic 스키마 + FastAPI 자동 검증
- XSS 방지: React의 기본 이스케이핑 활용, `dangerouslySetInnerHTML` 미사용

## 실행 방법

```bash
# 환경변수 설정 (선택 - 없으면 폴백 모드)
export ANTHROPIC_API_KEY=your-api-key

# 실행
chmod +x run.sh
./run.sh
```

또는 개별 실행:
```bash
# 백엔드
cd backend && source venv/bin/activate
python3 -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload

# 프론트엔드
cd frontend && npm run dev
```

## URL 구조
- `/shop` — 상품 목록
- `/shop/products/{id}` — 상품 상세 + 리뷰
- `/dashboard/tone` — 브랜드 톤 설정
- `/dashboard/reviews` — 리뷰 관리 + 대댓글 생성
- `/dashboard/history` — 히스토리
- `http://localhost:8000/docs` — API 문서 (Swagger)
