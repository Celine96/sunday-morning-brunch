# 리뷰 대댓글 생성 에이전트 — UI/UX 와이어프레임 (D-2 구조)

> 버전: Draft v0.2 | UI 형태: D-2 (대시보드 + FAB)
>
> **참고**: 본 문서는 구현 전 초기 설계안입니다. 최종 구현에서는 아래 변경이 반영되었습니다:
> - **대댓글 3개 후보 제공**: 단일 초안 → 후보 A/B/C 카드 선택 방식
> - **말투 변경 기능**: "재생성" 버튼 → 톤 드롭다운 (친근한/전문적인/감성적인)
> - **상품별 리뷰 그루핑**: 텍스트 입력 → 상품별 체크박스 선택 + 일괄 생성
> - **교환/환불 미언급 정책**: 대댓글에서 교환/환불 직접 언급 금지, 고객센터 유도
> - **사이드바 네비게이션**: 상단 탭 → 좌측 사이드바 (주황색 테마)
> - **쇼핑몰 디자인**: 히어로 배너, 카테고리 필터, cream/serif 톤 (TESER 참고)
> - **별점 필터**: 대시보드에 1~5점 + 긴급/주의 배지 필터 추가
> - **바로 등록**: confirm+publish 원클릭 처리 + 5초 실행 취소 토스트
> - **온보딩 페이지**: 루트(/) 경로에 서비스 소개 + CTA 페이지 추가

## 1. 전체 화면 구조

```
URL 구조:
/shop                        → 상품 목록 (샘플 쇼핑몰) + FAB
/shop/products/{id}          → 상품 상세 + 리뷰 + 대댓글 (샘플 쇼핑몰) + FAB
/dashboard/tone              → 브랜드 톤 설정 (에이전트 대시보드)
/dashboard/reviews           → 리뷰 목록 + 대댓글 생성 (에이전트 대시보드)
/dashboard/history           → 히스토리 (에이전트 대시보드)

FAB (플로팅 액션 버튼):
- 쇼핑몰 모든 페이지 우하단에 고정 (position: fixed)
- 클릭 시 미니패널 오버레이 오픈
- 쇼핑몰 레이아웃에 영향 없음 (z-index 오버레이)
```

## 4. 사용자 흐름 요약

### 흐름 A: 최초 설정 (대시보드)
```
에이전트 접속 → 톤 프로필 미설정 감지 → 톤 설정 페이지 안내
→ 브랜드명 + 키워드 입력 → (선택: 샘플 대댓글 입력)
→ 톤 프로필 생성 → 미리보기 확인 → 저장 → 리뷰 관리 페이지로 이동
```

### 흐름 B: 일상 사용 — 대시보드 경로
```
에이전트 접속 → 리뷰 관리 페이지 → 리뷰 입력 (단건/일괄)
→ 대댓글 생성 클릭 → 감성 분류 + 초안 생성 → 초안 확인/수정
→ 확정 → 게시 → 샘플 쇼핑몰에서 확인
```

### 흐름 C: 일상 사용 — FAB 경로 (단건 처리)
```
쇼핑몰 탐색 중 → 우하단 FAB 클릭 → 미니패널 오픈
→ 대댓글 미작성 리뷰 목록 표시 → 리뷰 선택
→ 단건 대댓글 생성 (감성분류 + LLM) → 초안 확인/수정
→ 확정 → 게시 → 쇼핑몰에서 즉시 확인
(선택) 미니패널 하단 "대시보드로 이동" 클릭 → 대시보드 페이지
```

### 흐름 D: 부정 리뷰 대응
```
리뷰 입력 (대시보드 또는 FAB) → 감성 "부정" 분류
→ 공감-사과-해결 구조 초안 생성
→ 운영자가 실제 대응방안 추가 수정 → 확정 → 게시
```

## 5. FAB 설계 상세

### 5-1. FAB 버튼 사양
- 위치: `position: fixed; bottom: 24px; right: 24px;`
- 크기: 56x56px (원형)
- 아이콘: 채팅 버블 아이콘
- z-index: 9999 (쇼핑몰 모든 요소 위)
- 애니메이션: 호버 시 확대, 미작성 리뷰 존재 시 뱃지 카운트 표시
- 클릭 시: 미니패널 토글 (열기/닫기)

### 5-2. 미니패널 사양
- 위치: FAB 버튼 좌상단으로 확장
- 크기: 약 360x500px (채널톡 유사)
- 오버레이: 쇼핑몰 레이아웃 영향 없음
- 헤더: 제목 + 닫기(×) 버튼
- 본문: 리뷰 목록 또는 대댓글 생성/편집 화면
- 푸터: "대시보드로 이동" 링크
- 전환: 리뷰 목록 ↔ 대댓글 생성 화면 간 내부 네비게이션

## 6. 반응형 설계 참고
- 모바일: 상품 카드 1열, 리뷰-대댓글 쌍 풀 width, FAB 미니패널 풀스크린 모드
- 태블릿: 상품 카드 2열, FAB 미니패널 360x500px
- 데스크톱: 상품 카드 3열, FAB 미니패널 360x500px
- 에이전트 대시보드: 최소 태블릿 이상 권장 (편집 영역 확보)
- FAB 버튼: 모든 뷰포트에서 동일 위치 (우하단 고정)
